# Design

## Armor Keeping Strategy

### Location

- On cloud `https://github.com/locchh/veronica/armors`

- Local path: `~/.claude/armors/`: NOT AVAILABLE NOW

### Naming convention

- Name of armor is the name of the folder
- Name of armor should NOT include spaces or special characters
- Name of armor should be in alphanumeric characters only
- Name of armor should NOT be longer than 50 characters

Example: `mod-01` or `mod_01`

### description.md convention

- One line, under 120 characters
- Shown in TUI card — NOT pulled to local

### Organization

```
<armor-name>/
├── description.md          # TUI display only — NOT pulled to local
├── CLAUDE.md
├── .mcp.json
├── .worktreeinclude
└── .claude/
    ├── settings.json
    ├── rules/
    ├── skills/
    ├── commands/
    ├── agents/
    └── output-styles/
```

## Claude folder structure

### Managed by Veronica (wipe + install on armor switch)

| No | Item | Location | Usage |
|---|---|---|---|
| 01 | `CLAUDE.md` | `./CLAUDE.md` or `.claude/CLAUDE.md` | Persistent instructions loaded into context every session |
| 02 | `settings.json` | `.claude/settings.json` | Enforced config: permissions, hooks, model, env vars |
| 03 | `rules/` | `.claude/rules/*.md` | Topic-scoped instructions; supports `paths:` frontmatter for conditional loading |
| 04 | `skills/` | `.claude/skills/<name>/SKILL.md` | Reusable workflows invokable via `/skill-name` or auto-invoked |
| 05 | `commands/` | `.claude/commands/*.md` | Legacy form of skills — still works, but `skills/` is preferred |
| 06 | `agents/` | `.claude/agents/*.md` | Specialized subagents with their own context, tools, and model |
| 07 | `output-styles/` | `.claude/output-styles/*.md` | Custom system-prompt styles (e.g. review mode, teaching mode) |
| 08 | `.mcp.json` | Project root | Team-shared MCP servers (GitHub, DBs, etc.) |
| 09 | `.worktreeinclude` | Project root | Gitignored files to copy into new git worktrees (e.g. `.env`) |

### Not touched

- `~/.claude/` — User-global scope, outside Veronica's jurisdiction
- `.claude/settings.local.json` — Personal overrides, auto-gitignored
- `.claude/agent-memory/` — Auto-generated by Claude at runtime; wiping it would lose user data

## Mechanism

### List armors

1. Fetch armor list from `https://github.com/locchh/veronica/armors`
2. Validate each armor's folder structure and naming convention
3. Read `description.md` from each armor (not saved locally)
4. Pass armor list to TUI

### Select armor (TUI)

1. Display each armor as a card with name and description (Yu-Gi-Oh style)
2. Navigate with arrow keys, confirm with `Enter`
3. Options available in TUI:
   - **Activate** — wipe current armor then pull selected armor
   - **Wipe** — wipe current armor without activating a new one

### Wipe armor

1. Read `.veronica-lock` to confirm an armor is active
2. Delete all managed files and folders:
   - `./CLAUDE.md`
   - `./.mcp.json`
   - `./.worktreeinclude`
   - `.claude/settings.json`
   - `.claude/rules/`
   - `.claude/skills/`
   - `.claude/commands/`
   - `.claude/agents/`
   - `.claude/output-styles/`
3. Clear `.veronica-lock`

### Pull armor

1. Fetch selected armor folder from cloud (excluding `description.md`)
2. Copy files to their target locations (mirror the armor folder structure)
3. Write `.veronica-lock` with the active armor name

### Download mechanism

Uses the GitHub Contents API to recursively fetch all files in an armor folder without cloning the full repo.

**Step 1 — Get file tree**

Fetch the armor folder contents:
`GET https://api.github.com/repos/locchh/veronica/contents/armors/<name>`

Returns a list of files and folders. For nested folders, recurse into each subdirectory.

**Step 2 — Download each file**

For each file (excluding `description.md`), fetch the raw content:
`GET https://raw.githubusercontent.com/locchh/veronica/main/armors/<name>/<path>`

**Step 3 — Write to disk**

Map each file path from the armor folder to its target location in the project:

| Armor path | Target path |
|---|---|
| `CLAUDE.md` | `./CLAUDE.md` |
| `.mcp.json` | `./.mcp.json` |
| `.worktreeinclude` | `./.worktreeinclude` |
| `.claude/` | `./.claude/` |

Create parent directories as needed before writing each file.

## Code Structure

```
veronica/
├── index.ts                  # CLI entry — launches TUI
├── src/
│   ├── tui/
│   │   ├── ArmorPicker.ts    # Card-based armor selector
│   │   └── animation.ts      # Deploy animation (capsule → unpack)
│   ├── armor/
│   │   ├── schema.ts         # Zod schema for armor metadata
│   │   ├── validator.ts      # Folder structure + naming check
│   │   ├── list.ts           # Enumerate armors + load description.md
│   │   ├── download.ts       # Fetch a single armor's files
│   │   ├── install.ts        # Write downloaded files to project
│   │   └── wipe.ts           # Remove all managed files
│   ├── github.ts             # GitHub Contents API + raw URL fetch
│   ├── lock.ts               # Read/write .veronica-lock
│   └── paths.ts              # Managed paths — single source of truth
├── scripts/
│   └── install.sh            # Bootstrap script (installs Bun + veronica CLI)
└── armors/                   # The repo itself is the cloud source
    └── mod-01/
        ├── description.md
        ├── CLAUDE.md
        └── .claude/...
```
